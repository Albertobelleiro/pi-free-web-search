# pi-free-web-search Design

## Goal
Build a free Pi package that gives agents high-quality web search and content extraction without paid APIs, using a hybrid pipeline: fast HTTP search first, browser automation fallback when quality requires it.

## Core principles
- Free to use: no paid APIs required.
- Quality-first: prefer better results over minimal implementation.
- Browser-aware: detect the user's default browser and search engine when possible.
- User-controlled automation: visible/headless/auto behavior should be configurable by the user.
- Professional Pi package: extension + skill + tests + CI + GitHub distribution.

## Architecture
1. Detection layer
   - Detect OS, default browser, browser family, executable path, and likely default search engine.
   - Support macOS and Linux.
2. Search orchestration layer
   - Resolve the active engine from config, detection, or fallback.
   - Try HTTP SERP extraction first.
   - Escalate to browser automation if result quality is poor and policy allows.
3. Content extraction layer
   - Fetch and extract article content using Readability.
   - Fallback to browser-rendered extraction for JS-heavy pages.
4. Pi integration layer
   - `free_web_search` tool
   - `free_fetch_content` tool
   - `/free-search-info` command
   - `/free-search-test` command
   - skill that teaches the agent how to use the tools well.

## Supported behavior for v0.1
- macOS + Linux
- Default browser detection
- Search engine detection for Safari, Chromium-family browsers, and partial Firefox support
- Search engine adapters for Google, Bing, DuckDuckGo, Brave, and generic templates
- Browser fallback using Playwright
- Readability-based extraction
- Tests, smoke test, and CI

## Explicit non-goals for v0.1
- Windows support
- YouTube / PDF / GitHub cloning parity with pi-web-access
- Full TUI curator UI
- Persistent cross-session result storage
